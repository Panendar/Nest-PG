## Objective

Help the User persona find suitable PG options with less time and effort by moving from fragmented offline discovery to a single, trusted discovery-to-contact journey. This module exists so PG seekers can search relevant options, compare them confidently, and contact the right owner quickly.

## Success Outcomes

- Week-over-week growth in user-to-owner contact requests from this module.
- Higher share of active users who move from search activity to owner contact within the same journey.
- Reduced user drop-off during PG discovery and shortlisting compared with the current baseline.
- Improved user confidence in decision-making, reflected by stronger engagement with listing comparison and contact actions.
- [MISSING: Numeric targets and time-bound thresholds for the outcomes above are not defined in the source module documentation.]

## Must-Have Features

- Users can search PG options by city and nearby radius to quickly find relevant listings.
- Users can view enough listing information to evaluate suitability before contacting an owner.
- Users can compare PG options to narrow choices with confidence.
- Users can contact owners directly from the module after authentication.
- Users can rely on current listing availability/status while making decisions.

## Optional Features

- Save and revisit shortlisted listings in a future release.
- Guided recommendations to help users prioritize listings based on their preferences.
- Follow-up reminders for users who started searching but have not yet contacted an owner.

## Out of Scope

- Owner-side listing creation, editing, media management, and availability management.
- Admin moderation, governance, and marketplace operations.
- Anonymous visitor access to full listing details and direct owner contact.
- Any workflow whose primary outcome is platform administration rather than user discovery-to-contact.
