## Summary

Search By City lets authenticated users enter a city name and view PG listings relevant to that city. This feature reduces the time and effort users spend on fragmented offline discovery by giving them a focused starting point for shortlisting options. It matters because city-based discovery is a core step in the module's discovery-to-contact journey.

## User Story

As a User, I want to search PG listings by city so that I can quickly find relevant options in my preferred location and decide which owners to contact.

## Acceptance Criteria

- The user can enter a city and trigger a search from the user search screen.
- After search is triggered, the user sees a results view that is clearly tied to the entered city.
- The user can update the city and run a new search without leaving the search experience.
- If no listings are available for the entered city, the user sees an explicit empty-state message with a clear next action.
- If the city input is empty when the user tries to search, the user sees a validation message that explains what to correct.
- From city search results, the user can open a listing to continue evaluation.

## Priority

High - City-based search is a primary discovery path and foundational to the module's core user outcome.

## Dependencies

none
