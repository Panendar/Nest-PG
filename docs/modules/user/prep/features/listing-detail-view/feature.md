## Summary

Listing Detail View lets authenticated users open a PG listing and review the information needed to judge fit before taking the next step. It matters because users need confidence in a specific option, not just a list of options, to avoid wasted effort and wrong contacts. This feature supports faster shortlisting and better discovery-to-contact decisions.

## User Story

As a User, I want to view complete details of a selected PG listing so that I can decide whether it matches my needs before I contact the owner.

## Acceptance Criteria

- From search results, the user can open a listing detail view for a selected listing.
- The detail view shows key evaluation information for that listing, including availability status and core listing details.
- The user can return to the previous results view without losing their search context.
- If a listing is no longer available, the user sees a clear message and can continue browsing other listings.
- From the listing detail view, the user can choose a next action to continue evaluation (for example, compare or contact owner).
- Access to full listing detail view is available only to authenticated users.

## Priority

High - Listing detail review is a core decision point between discovery and contact in the primary user journey.

## Dependencies

none
