## Summary

Compare Listings allows authenticated users to evaluate multiple PG options side by side before deciding what to shortlist or contact. It matters because users need a clear way to judge trade-offs across options instead of switching back and forth between separate listing pages. This feature improves confidence and reduces decision time in the discovery journey.

## User Story

As a User, I want to compare multiple PG listings in one view so that I can quickly identify the best option for my needs and take the next action with confidence.

## Acceptance Criteria

- The user can select at least two listings from search results and open a compare view.
- In compare view, the user sees the selected listings presented in a consistent side-by-side structure.
- The user can remove a listing from compare view and the comparison updates immediately.
- The user can return from compare view to search results without losing their current search context.
- From compare view, the user can choose a next action on a listing (for example, view details or contact owner).
- If fewer than two listings are selected, the user sees clear guidance to add more listings before comparing.

## Priority

High - Comparison is a core decision aid that directly supports faster shortlisting and stronger discovery-to-contact conversion.

## Dependencies

none
