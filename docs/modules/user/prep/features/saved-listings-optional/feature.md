## Summary

Saved Listings Optional lets users save PG options they want to revisit later instead of relying on memory or repeated searching. It matters because users often need time to compare multiple suitable options before deciding, and a saved list helps them keep track of promising listings as they narrow choices. This feature supports more deliberate decision-making in the discovery journey.

## User Story

As a User, I want to save listings I am interested in so that I can revisit them later and compare my options before contacting an owner.

## Acceptance Criteria

- The user can save a listing from a search or detail view.
- The user can see a list of saved listings they can revisit later.
- The user can remove a listing from their saved list.
- The user can open a saved listing and return to its details without starting a new search.
- The user can clearly tell which listings are already saved while browsing.
- If there are no saved listings, the user sees a clear empty state with a prompt to continue browsing.

## Priority

Low - This is useful convenience functionality, but the core discovery and contact journey can still work without it.
