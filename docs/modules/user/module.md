### Purpose

This module is the authenticated user experience for PG seekers to discover and evaluate PG options in one place. It solves the current fragmented process where students rely on manual, offline exploration across multiple accommodations. The scope is discovery-to-contact for users: search by city or nearby radius, compare listings, and reach owners directly from the platform.

### Users

Primary persona: User. Supporting funnel relationship: Visitor may convert into User before using this module deeply. Users are trying to find suitable PG options faster, shortlist confidently, and contact the right owner without repeated offline effort.

### Business Rationale

This module is being built now because user-side discovery-to-contact is the product's primary value outcome and key to reducing search friction. Without it, the platform cannot deliver its core promise of centralized, local PG comparison and direct owner reach, which weakens adoption and top-of-funnel conversion into meaningful activity. It directly addresses the risk of users continuing to depend on fragmented, inefficient offline discovery.
